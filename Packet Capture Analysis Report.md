#####                                **Packet Capture Analysis Report**







###### **1.Objective**



To capture live network packets and identify common network protocols using Wireshark.







###### **2.Tool Used**



* Wireshark







###### **3.Capture Details**



* Interface Used: Wi-Fi
* Capture Duration: 1 minute
* Traffic Generated: Web browsing and ping test







###### **4.Protocols Identified**



&nbsp;**1. DNS (Domain Name System)**



* Used to resolve domain names into IP addresses.
* Observed queries for google.com.
* Uses UDP port 53.



&nbsp;**2. TCP (Transmission Control Protocol)**



* Provides reliable communication.
* Used in web browsing sessions.
* Observed TCP handshake (SYN, SYN-ACK, ACK).



&nbsp;**3.HTTP (Hypertext Transfer Protocol)**



* &nbsp;Uses TCP port 80
* &nbsp;Request Method: GET







###### **5.Conclusion**



The packet capture successfully identified multiple protocols including DNS, TCP, ICMP, and TLS. 

The analysis demonstrates understanding of network traffic and protocol behavior.
